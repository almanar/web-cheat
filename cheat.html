#img

alt

If:

  • an image does not exist
  • the format is not supported by the browser (support list)

then alt the browser can show the alt text instead of the image.

As of 2013, chrome replaces a missing image with a dummy image, therefore ignoring alt therefore don't rely on alt being shown in your app logic: alt should only be shown in case of error. If an image should not present, replace it with something else explicitly.

fig + figcaption is good practice in HTML5. figcaption has no special display, it is only semantic and the real work must be done with css. Browsers may however place it in a sensible place such as the bottom if no style data is found.

#pre

White spaces are taken into consideration when rendering.

I think that this is a bad tag because it implies style rather than meaning and style should be left for CSS, and it can in this case via white-space:pre. The only difference is that pre ignores the first character if it is a newline, while white-space:pre does not.
